## Drag-Drop Interaction Flow

### 搭建操作
1. 玩家 mousedown 在材料栏的 Board 上 → Board 跟随鼠标
2. 拖动到缺口格位上方 → 格位高亮 (可放置指示)
3. mouseup 在有效格位上 → Board 放置成功
   - materials.board--
   - 格位显示 Board 图形
   - stress 条实时更新 (显示预估)
4. mouseup 在无效位置 → Board 返回材料栏

### 绑绳操作
1. mousedown 在材料栏的 Rope 上 → Rope 跟随鼠标
2. 拖动到两块相邻已放置 Board 之间 → 连接线高亮
3. mouseup → Rope 绑定成功
   - materials.rope--
   - 两块 Board 间显示绳索连接
   - segment_strength 更新

### 锚楔操作
1. mousedown 在材料栏的 Wedge 上 → Wedge 跟随鼠标
2. 拖动到已放置 Board 上 → Board 高亮
3. mouseup → Wedge 锚定成功
   - materials.wedge--
   - Board 旁显示楔子标记
   - 该段 wind 减免生效

### 排序操作
1. mousedown 在人员卡片上 → 卡片浮起
2. 拖动到队列中其他位置 → 队列实时重排
3. mouseup → 新顺序确认

## Feedback Channels

### 结构受力颜色
- stress < 30: 绿色 #4CAF50
- stress 30–60: 黄色 #FFC107
- stress 60–85: 橙色 #FF9800
- stress ≥ 85: 红色 #F44336 (闪烁)
- 坍塌时: 全屏震动 + 红色闪烁

### Wind 偏移
- Wind 条颜色随风值变化
- 缺口区域显示风向动画 (粒子或线条)
- 高 wind 时缺口格位轻微抖动

### 材料损耗
- 材料栏实时更新数量
- 材料用尽时该材料图标变灰 + 不可拖拽

### 通过进度
- crossing_progress 数字实时更新
- 人员通过时卡片从队列滑向缺口
- 成功通过: 卡片移到对面 (绿色标记)
- 坠落: 卡片下沉消失 (红色标记)

## Interaction States

| 状态 | 可操作 | 说明 |
|------|--------|------|
| VIEWING_GAP | 拖放材料到缺口 | 观察阶段，放置材料 |
| BUILDING | 拖放材料、绑绳、锚楔 | 持续搭建 |
| ORDERING | 拖拽人员排序 | 安排通行顺序 |
| CROSSING | 无 (自动) | 系统逐人结算 |
| COLLAPSE | 无 | 坍塌动画 + GAME_OVER |
| PHASE_COMPLETE | 无 | 短暂庆祝 → 下一 Phase |

## Forbidden UI

- 不允许做终章剧情画面
- 不允许只用"让谁先过"文字按钮列表替代拖拽排序
- 不允许用纯文字选项替代材料放置操作
- 不允许隐藏 stress/wind 数值

## Acceptance Rules

- 首屏必须让玩家看到缺口区域和至少一个可拖拽的材料
- 拖放操作必须产生即时可见反馈 (颜色/位置/数值变化)
- 每次操作反馈必须能追溯到 Required State 的变化
- 不得只靠随机事件文本或普通选择按钮完成主循环
